为什么要做这个限制

这是因为Android中的UI控件不是线程安全的，如果不做这个限制，在多个线程同时操作UI时，就会导致UI的状态不可预期。就是我们平时所说的线程安全的问题。线程不安全我们可以通过加锁机制来解决，但是加锁会降低UI访问的效率，因此Android并没有采用这种处理方法，而是采用更加简单而且高效的方法，单线程模型来处理UI操作。也就是我们为什么要使用Handler来更新UI的原因啦。
